You are an expert C# engineer working on a WPF 6/45 Lotto number generator.
Add a filtering module that enforces the following rules for a 6-number combination (1–45, ascending, no duplicates):

추가 필터
1. 홀짝 비율
“홀/짝이 0개 또는 1개인 조합”은 제외한다.
즉, oddCount ∈ {0,1} 또는 evenCount ∈ {0,1} 이면 fail.
6개 조합에서는 유효 범위: 2 ≤ oddCount ≤ 4.
예: 1,3,5,7,9,11(짝 0개) 제외 / 1,2,4,6,8,10(홀 1개) 제외.

2. 10단위(데케이드) 그룹 과밀
같은 10단위(0–9, 10–19, 20–29, 30–39, 40–45)에 3개 이상 몰리면 제외한다.
즉, 각 decade당 개수 ≤ 2 이어야 한다.
예: 1,2,5,7,14,25 (0–9에 4개) 제외 / 22,25,26,29,30,36 (20대 4개) 제외.

3. 기존 1등 당첨 조합 제외
프로그램 초기 실행 시, URL(https://www.dhlottery.co.kr/common.do?method=getLottoNumber&drwNo=1)을 참고하여 회차별 당첨 데이터를 불러온다.
불러온 데이터는 Cache CSV 파일로 별도 저장한다.
프로그램 시작할 때마다 최신 회차가 있는지 확인한 후, 캐시 파일을 업데이트한다.
필터링 로직에서는 캐시에 저장된 과거 1등 당첨번호 조합과 동일한 조합을 제외한다.
**URL의 drwNo 수치는 회차를 나타냄

4. 연속수(pair) 허용 범위
연속된 숫자 쌍(x,x+1)은 최대 1쌍만 허용한다. 2쌍 이상이면 제외.
예: 1,2,12,17,30,35 → 1쌍(1–2) 포함 / 11,12,21,25,44,45 → 2쌍(11–12, 44–45) 제외.

5. 같은 끝자리(일의 자리) 규칙
같은 끝자리 숫자가 없거나,
정확히 2개짜리 1쌍만 있거나,
정확히 2개짜리 2쌍만 있는 경우만 허용한다.
즉, 어떤 끝자리도 3개 이상 나오면 제외하고, 2개 짜리 끝자리 그룹의 개수는 0, 1, 2만 허용.
예(허용):
1,3,14,25,36,39 (끝자리 전부 다름)
1,3,13,25,36,39 (끝자리 3이 2개 → 2개짜리 1쌍)
1,3,13,25,35,39 (끝자리 3이 2개, 5가 2개 → 2개짜리 2쌍)
